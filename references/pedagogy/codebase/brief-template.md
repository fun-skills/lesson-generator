# 内容简报模板（代码库课程差异教学法）

> 何时阅读：复杂代码库需要并行或分段生成课程内容时阅读。

简报不是 HTML 模块说明，用于指导写某一节或几节 `lesson` 数据，让每节可独立生成，不必反复读取整个代码库。后续生成 `data.js` 时不重新读完整代码库，来源路径必须可追溯，不改写真实代码。

将每份简报保存到：

```text
course-name/briefs/0N-slug.md
```

## 第 N 节：[标题]

### 教学目标

- 本节要让学习者理解什么？
- 这对他们指挥 AI、调试或做架构决策有什么帮助？

### 用户入口

从学习者见过的哪个动作开始？

```text
例如：点击"生成报告"后，数据如何从前端进入后端。
```

### 核心隐喻

使用一个具体、新鲜、贴合概念的隐喻。不要重复前面模块用过的隐喻。

### 真实代码片段

列出本节会用到的真实代码，包含文件路径和行号。不要要求后续写作代理重新读整个代码库。

```text
文件：src/example.ts，第 12-24 行
用途：解释请求如何发出

[粘贴真实代码]
```

### 推荐 body blocks

```text
- p：开场，连接用户操作
- architecture：介绍角色
- flow：展示数据流
- code-translation：解释真实代码
- actor-chat：让组件对话
- debug-case：讲一个常见故障
- insight：总结可迁移的经验
```

只选择适合本节的 block，不要机械填满。

### 必须出现的互动

明确本节必须包含哪些互动（如至少一个 `code-translation`、一个 `actor-chat` 或 `flow`）。

### 测验方向

写 2-4 道应用型题目的方向。优先：用户反馈某症状先查哪里、新功能应该放在哪一层、数据路径如何走、应该怎样提示 AI 修改。

### 闪卡术语

列出本节最重要的 3-5 个术语或判断。每张卡背面必须用白话解释。

### sources

列出本节 sources（真实文件路径和行号）。

```text
src/example.ts:12-24
src/service/api.ts:8-30
README.md
```

### 衔接

- 上一节讲了什么？
- 下一节会接到哪里？
- 哪些术语或角色命名要保持一致？
