# 内容风格（共同教学法）

> 何时阅读：写课程正文和 body 时阅读。所有课程都适用。

## 正文精简

- 不写长篇大论。每个文本 block 最多 2-3 句话。
- 超过这个长度就拆成多个 block 或换成视觉 block。
- 保持信息密度高以支持反复学习，但不要堆砌装饰。

## 先示例后抽象

- 先给具体示例，再抽象出概念。
- 目标用简短要点，解释用可读段落。

## 一个 block 一个概念

- 不要把多个概念塞进一个 block。
- 一个段落同时讲用户行为、组件职责、数据流、调试方法时，拆成多个 block。

## 编写规范

- 课程数据保持 JSON 可序列化；UI 字符串用双引号或模板字面量；不要在单引号字符串中放未转义的撇号。
- 只写 `data.js`，不写自由 HTML，不在 `text` / `analysis` / `explanation` 里塞 HTML 字符串。
- 内容应该是结构化数据，渲染由 class-viewer 负责。
